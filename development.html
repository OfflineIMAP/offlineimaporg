Development

Mailing list

The user discussion, development and all exciting stuff take place on the mailing list. It's email address is offlineimap-project@lists.alioth.debian.org. You do NOT need to be subscribed to send emails to this list (But don't forget to ask for a 'CC to you if you want to see the replies).

Reporting bugs

Bugs and issues should be reported to the mailing list. Do make sure you report the OfflineIMAP version you are using as well as the brand of IMAP server involved. If you are using a stable version that came with your distribution, we would appreciate testing to make sure the bug hasn't been fixed already.

How to create a debug log

If you get a crash with offlineimap, we would appreciate if you send in a debug log, so we can analyze the reasons for the crash. You can create a debug log by appending "-1 -d imap -l debug.log" to your offlineimap invocation. It would be great if you could make that log available to us so we can analyze it and prevent the situation from happening again (or at least spit out nicer error messages). Take note that the debug.log could contain passwords and email contents though. So either remove those or make the log available to one of the developers in private.

Get the Source code

Our git repository home page can be found on its github home. Grab the current code with a git clone git://github.com/nicolas33/offlineimap.git.

There are several branches. Master represents the current stable branch or what will be the next stable release. Next is where development happens. This is what will be merged into Master for the next major release. Finally PU is a feature branch which hosts crazy features that might or might not make it into some release. This branch could easily be reset at any point in time.
